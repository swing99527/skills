# 泛能网光伏运维后台数据采集 Skill

## 概述

该 Skill 用于自动化登录泛能网（fanneng.com）光伏运维后台，采集各电站的光伏发电数据和设备运维状态。

## 适用场景

- 用户需要查看光伏电站的发电量数据
- 用户需要统计逆变器设备的运行状态（正常/停机/离线/告警）
- 用户需要跨多个电站汇总运维数据
- 定期巡检数据采集报告

## 系统信息

- **平台地址**: https://web.fanneng.com/
- **登录地址**: https://authentication-center-new.ennew.com/login?appid=fnw-auth-manage&redirect=https%3A%2F%2Fweb.fanneng.com%2F&terminalType=PC-WEB-CHROME&tenantPageName=fanneng&sdkVersion=0.2.6&withTempAuthCode=true&checkWhiteUser=1&isDirectPage=1&logout=1
- **平台技术**: 基于 iframe 嵌套的 SPA 应用，使用新奥数能科技平台构建

## 操作流程

### Step 1: 登录

1. 打开登录地址（上方完整 URL）
2. 输入手机号到手机号输入框
3. 输入密码到密码输入框
4. **重要**: 页面底部有一个用户协议勾选框，必须点击勾选才能登录
5. 点击登录按钮
6. 等待跳转到 https://web.fanneng.com/

> [!IMPORTANT]
> 登录页面底部的用户协议 checkbox 可能不太容易定位，它位于登录按钮下方，大约在页面 y=690 的位置。必须勾选才能成功登录。

### Step 2: 识别电站列表

- 页面左上角显示当前电站名称（如"天津集通新能源有限公司"）
- 点击电站名称会展开下拉菜单，显示所有可用电站
- 当前已知电站：
  1. **滁州集通新能源有限公司** — 子站: 中通快递滁州转运中心4.23891MW光伏项目
  2. **天津集通新能源有限公司** — 子站: 中通中快供应链屋顶1.1MW分布式光伏

### Step 3: 导航到光伏运行页面

路径: **左侧菜单 → 项目经营 → 生产分析 → 光伏运行**

> [!NOTE]
> 页面使用 iframe 嵌套，菜单项在 iframe 内部。点击后需要等待页面内容加载。

使用 MCP chrome-devtools 操作时的参考 UID 路径:
```
menuitem "项目经营" → 展开后 → menuitem "生产分析" → 展开后 → menuitem "光伏运行"
```

### Step 4: 采集光伏监视 Tab 数据

光伏运行页面有 4 个 Tab:
- **光伏监视**（默认选中）— 核心发电数据
- **逆变器** — 设备状态明细
- 并网柜
- 气象站

**光伏监视 Tab 包含以下关键数据:**

| 字段 | 说明 |
|---|---|
| 当日发电量 | 当天累计发电量 (kWh 或 万kWh) |
| 装机容量 | 电站总装机容量 (kW) |
| 当月发电量 | 本月累计发电量 (万kWh) |
| 首次并网时间 | 电站首次连网日期 |
| 当年发电量 | 本年度累计发电量 (万kWh) |
| 经纬度 | 电站地理坐标 |
| 当日等效利用小时数 | 发电时长折算 (h) |
| 倾斜角度 | 光伏板倾斜角度 (°) |
| 当日发电效率 | 当天发电效率 (%) |
| 累计减排CO₂ | 累计碳减排量 (t) |
| 电站地点 | 具体地址 |

页面顶部有**电站名称下拉框**，用于选择当前公司下的子电站。

### Step 5: 采集逆变器 Tab 数据

点击"逆变器" Tab，页面显示所有逆变器设备卡片。

**顶部状态汇总栏**显示:
- 全部 (X台)
- 正常 (X台) 
- 离线 (X台)
- 停机 (X台)
- 告警 (X台)

**每个逆变器卡片包含:**
- 设备名称（如"华为逆变器NB1"）
- 额定功率 (kW)
- 当前状态（停机/正常/离线/告警）
- 实时参数:
  - 日实际发电量 (kWh)
  - 日等效利用小时数 (h)
  - 日发电效率 (%)
  - 有功功率 (kW)
  - 机内空气温度 (°C)

> [!NOTE]
> 逆变器列表可能需要**向下滚动**才能看到所有设备。天津站有 10 台，滁州站有 43 台。

### Step 6: 切换电站并重复采集

1. 点击左上角电站名称
2. 在下拉菜单中选择下一个电站
3. 等待页面刷新（可能需要重新导航到 光伏运行 页面）
4. 重复 Step 3-5

> [!WARNING]
> 切换电站时，系统可能会重定向到登录页面。如果发生这种情况，需要重新执行登录流程。

### Step 7: 汇总数据

最终输出应包含:

1. **发电量汇总表**: 各电站的日/月/年发电量
2. **设备状态汇总表**: 各电站正常/停机/离线/告警设备数量
3. **注意事项**: 如果是夜间采集，所有逆变器显示"停机"是正常现象

## 数据采集注意事项

1. **夜间采集**: 太阳能电站夜间所有逆变器均为停机状态，有功功率为 0.00 kW，这是**正常**现象
2. **白天采集**: 建议在 9:00-16:00 时段采集，可获取实时运行数据
3. **数据单位**: 发电量可能显示为 kWh 或 万kWh，注意单位转换
4. **页面加载**: 由于使用 iframe，页面切换后需要等待 2-3 秒让数据加载完成

## 推荐工具

- **MCP chrome-devtools**: 适合精确操作菜单和读取 DOM 数据
- **browser_subagent**: 适合完整的端到端自动化流程
- 两者结合使用效果最佳：先用 browser_subagent 登录，再用 chrome-devtools 精确采集
